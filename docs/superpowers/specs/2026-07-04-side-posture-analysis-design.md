# 옆모습 자세 분석 MVP 아키텍처 설계

## 목표

자세 알리미 MVP는 스마트폰을 웹캠처럼 연결하고, 노트북 브라우저에서 옆모습을 실시간으로 분석해 부위별 자세 경고를 보여주는 웹 앱이다.

사용자는 카메라를 켜면 자신의 영상 위에 뼈대와 핀포인트 오버레이를 본다. 문제가 있는 부위는 주황색으로 강조한다. 오른쪽 패널에는 전체 점수와 부위별 경고 사유를 보여준다.

이 앱은 의학적 진단 도구가 아니다. 웹캠 기반 자세 참고 도구로 동작한다.

## MVP 범위

- Vite + TypeScript + 순수 DOM/Canvas로 브라우저 단독 앱을 만든다.
- 백엔드는 두지 않는다.
- 영상은 저장하지 않는다.
- MediaPipe Pose Landmarker로 실시간 관절 좌표를 추출한다.
- 옆모습 단일 카메라 기준으로 분석한다.
- 실제 관절점과 가상 해부학 포인트를 구분해서 그린다.
- 목, 경추 추정, 척추 라인 추정, 요추 추정, 상체 기울어짐을 분석한다.
- 다리꼬임과 골반 비틀림은 이번 MVP에서 정밀 판정하지 않는다.

## MVP 제외 범위

- 정면 모드 분석
- 다리꼬임 정밀 판정
- 골반 비틀림 정밀 판정
- 자세 기록과 통계
- 소리 알림
- 사용자별 캘리브레이션
- 의료 수준의 해부학 추적

다리꼬임과 골반 비틀림은 나중에 정면 모드를 추가할 때 다룬다. 현재 구조에는 `AnalysisMode` 개념을 남겨서 `side`와 `front`를 분리할 수 있게 한다.

## 기술 선택

MediaPipe Pose Landmarker를 사용한다. 공식 문서 기준으로 Pose Landmarker는 이미지 좌표와 3D 월드 좌표의 body pose landmarks를 반환한다. 브라우저에서는 `@mediapipe/tasks-vision`을 사용한다.

실제 경추, 흉추, 요추 뼈 위치를 직접 검출하지는 않는다. 따라서 경추, 척추, 요추 포인트는 MediaPipe 관절점으로부터 계산한 가상 해부학 포인트로 다룬다.

## 모듈 구조

### `camera`

카메라 권한과 비디오 스트림을 관리한다.

- 카메라 장치 목록 조회
- 기본 카메라 선택
- 스트림 시작과 중지
- 비디오 엘리먼트에 스트림 연결
- 권한 거부와 장치 없음 상태 전달

### `vision`

MediaPipe Pose Landmarker를 감싼다.

- 모델 파일 로딩
- `VIDEO` 모드 초기화
- 프레임별 pose landmark 추론
- 추론 실패와 모델 로딩 실패 상태 전달

### `anatomy`

실제 관절점에서 분석용 기준점을 만든다.

- 왼쪽/오른쪽 중 옆모습에서 더 잘 보이는 측면 선택
- 귀, 어깨, 골반, 무릎, 발목 관절점 정규화
- 경추 추정점 계산
- 상부 척추 추정점 계산
- 중부 척추 추정점 계산
- 요추 추정점 계산
- 골반 기준점 계산

가상 해부학 포인트는 화면과 데이터 모델에서 명확히 `inferred`로 표시한다.

### `analysis`

자세 상태와 점수를 계산한다.

- 목 전방 이동 분석
- 경추 기울기 분석
- 상체 기울어짐 분석
- 척추 라인 무너짐 분석
- 요추 기준 분석
- 부위별 `ok`, `warning`, `danger`, `unstable` 상태 계산
- 전체 자세 점수 계산
- 추적 신뢰도 계산

### `smoothing`

프레임별 흔들림을 줄인다.

- 최근 N프레임 이동 평균
- 부위별 상태가 너무 빠르게 깜빡이지 않도록 안정화
- 추적 불안정 프레임은 점수에 무리하게 반영하지 않음

### `overlay`

비디오 위 캔버스 렌더링을 담당한다.

- 비디오 크기와 캔버스 크기 동기화
- 실제 관절점 렌더링
- 가상 해부학 포인트 렌더링
- 문제 부위 주황색 강조
- 정상 부위 녹색 또는 중립색 표시
- 추적 불안정 부위 흐림 처리

### `ui`

화면 상태와 사용자 조작을 담당한다.

- 시작/중지 버튼
- 카메라 선택
- 전체 점수 표시
- 부위별 경고 카드
- 추적 상태 메시지
- 오류 상태 표시

## 데이터 흐름

```text
카메라 프레임
→ MediaPipe Pose Landmarker
→ 실제 관절점 정규화
→ 옆모습 기준 측면 선택
→ 가상 해부학 포인트 계산
→ 부위별 분석
→ 점수와 경고 계산
→ 비디오 캔버스 오버레이 렌더링
→ 오른쪽 패널 갱신
```

모든 처리는 브라우저 안에서 끝난다. 프레임 이미지는 서버로 보내지 않는다.

## 주요 데이터 모델

```ts
type AnalysisMode = "side";

type Severity = "ok" | "warning" | "danger" | "unstable";

type TrackedPoint = {
  id: string;
  x: number;
  y: number;
  z?: number;
  visibility: number;
  source: "mediapipe" | "inferred";
};

type BodyPartStatus = {
  part: "neck" | "cervical" | "spine" | "lumbar" | "trunk";
  severity: Severity;
  score: number;
  message: string;
  pointIds: string[];
};

type PostureAnalysis = {
  mode: AnalysisMode;
  overallScore: number;
  confidence: number;
  statuses: BodyPartStatus[];
  points: TrackedPoint[];
};
```

## 분석 기준

### 옆모습 기준 측면 선택

왼쪽 귀/어깨/골반과 오른쪽 귀/어깨/골반의 `visibility`를 비교한다. 더 안정적으로 보이는 쪽을 분석 기준으로 삼는다.

선택된 측면의 주요 관절점 visibility가 `0.55`보다 낮으면 해당 부위는 `unstable`로 처리한다. 귀, 어깨, 골반이 모두 불안정하면 해당 프레임 전체를 `unstable`로 처리한다.

### 목 전방 이동

귀가 어깨보다 앞쪽으로 얼마나 이동했는지 본다. 사람과 카메라 거리 차이를 줄이기 위해 어깨-골반 길이 같은 몸 기준 길이로 정규화한다.

### 경추 기울기

귀와 어깨를 잇는 벡터가 수직 기준에서 얼마나 벗어났는지 본다. 각도가 커질수록 주의 또는 위험으로 올린다.

### 척추 라인 추정

경추 추정점, 상부 척추 추정점, 중부 척추 추정점, 요추 추정점을 연결한다. 이 라인이 한쪽으로 크게 무너지면 척추 라인 경고를 낸다.

### 요추 기준

요추 추정점과 골반 기준점의 관계를 본다. 상체 기울어짐과 함께 무너지면 요추 경고를 강화한다.

### 상체 기울어짐

어깨와 골반을 잇는 선이 수직 기준에서 얼마나 기울었는지 본다. 앞쪽으로 기울수록 점수를 낮춘다.

## 점수 계산

부위별 점수는 0~100으로 계산한다.

- 80~100: 양호
- 60~79: 주의
- 0~59: 위험
- 추적 불안정: 점수 계산에서 제외하거나 낮은 신뢰도로 표시

전체 점수는 목, 경추, 척추, 요추, 상체 점수를 가중 평균한다. MVP 초기값은 목과 상체 기울어짐에 더 높은 가중치를 둔다.

## 화면 구성

화면은 부위별 경고 우선형으로 만든다.

- 왼쪽: 카메라 영상과 캔버스 오버레이
- 오른쪽: 전체 점수, 부위별 경고, 추적 신뢰도
- 하단 고정 툴바: 시작/중지, 카메라 선택

오버레이 색상 의미는 다음과 같다.

- 파란색 또는 회색: MediaPipe 실제 관절점
- 흰색 또는 연한 회색: 정상 연결선
- 초록색: 정상 상태의 주요 기준점
- 주황색: 주의 또는 위험 부위
- 흐린 회색: 추적 불안정

## 오류 처리

### 카메라 권한 거부

브라우저 카메라 권한을 허용해야 한다는 메시지를 보여준다.

### 카메라 장치 없음

연결된 카메라가 없다는 메시지를 보여준다.

### 모델 로딩 실패

네트워크 또는 모델 파일 로딩 실패 메시지를 보여주고 재시도 버튼을 제공한다.

### 사람이 감지되지 않음

영상 안에 사람이 보이지 않는다는 메시지를 보여준다.

### 옆모습 추적 불안정

귀, 어깨, 골반 기준점이 충분히 보이지 않으면 판정을 멈추고 카메라 위치를 조정하라는 메시지를 보여준다.

### 성능 저하

프레임 처리 시간이 길어지면 분석 주기를 낮춘다. 화면은 계속 유지한다.

## 테스트 기준

### 단위 테스트

Vitest로 계산 로직을 테스트한다.

- 각도 계산
- 거리 정규화
- 측면 선택
- visibility 기준 처리
- 가상 해부학 포인트 계산
- 부위별 severity 계산
- 전체 점수 계산

### 통합 테스트

실제 카메라 없이 샘플 landmark 배열을 넣어 `anatomy`와 `analysis` 흐름을 검증한다.

### 수동 검증

브라우저에서 다음을 확인한다.

- 카메라 권한 요청이 정상 동작한다.
- 영상 위 캔버스가 정확히 겹친다.
- 사람이 움직이면 포인트가 따라온다.
- 문제 부위가 주황색으로 강조된다.
- 카메라를 가리면 추적 불안정 상태가 표시된다.
- 영상이 저장되거나 외부로 전송되지 않는다.

## 확장 방향

### 정면 모드

나중에 `AnalysisMode`에 `front`를 추가한다.

정면 모드에서는 다리꼬임, 골반 좌우 높이 차이, 골반 비틀림 추정, 어깨 좌우 높이 차이를 분석한다.

### 캘리브레이션

사용자가 바른 자세를 1회 등록하면, 고정 기준값 대신 개인 기준값을 사용한다.

### 기록과 알림

점수 기록, 통계, 소리 알림, 화면 알림은 MVP 이후에 추가한다.
